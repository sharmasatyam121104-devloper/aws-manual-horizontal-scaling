## 📖 Overview

Horizontal scaling is the process of increasing application capacity by adding multiple servers instead of upgrading a single server.

### Architecture

```text
Users
  |
  v
Route 53 (Domain)
  |
  v
Application Load Balancer
   /      |      \
EC2-1   EC2-2   EC2-3
```

Traffic is distributed across multiple EC2 instances, improving availability, scalability, and reliability.

---

# 🚀 Step 1: Create a Security Group

1. Open AWS Console.
2. Navigate to **EC2 → Security Groups**.
3. Click **Create Security Group**.
4. Provide a name and description.

### Inbound Rules

| Type  | Port | Source   |
| ----- | ---- | -------- |
| SSH   | 22   | My IP    |
| HTTP  | 80   | Anywhere |
| HTTPS | 443  | Anywhere |

5. Click **Create Security Group**.

---

# 🚀 Step 2: Launch Your First EC2 Instance

1. Navigate to **EC2 → Instances**.
2. Click **Launch Instance**.
3. Enter an instance name.
4. Select **Ubuntu Server**.
5. Choose an instance type.
6. Create or select an SSH Key Pair.
7. Attach the Security Group created earlier.
8. Click **Launch Instance**.

---

# 🚀 Step 3: Connect to the Instance

### Linux / Mac

```bash
ssh -i key.pem ubuntu@PUBLIC_IP
```

### Windows

Use:

* PuTTY
* MobaXterm
* Windows Terminal

```bash
ssh -i key.pem ubuntu@PUBLIC_IP
```

# 🚀 Step 4: Configure the Server

Update packages:

```bash
sudo apt update
sudo apt upgrade -y
```

Install the required software and runtime for your application.

Example:

```bash
sudo apt install nodejs npm -y
```

Verify installation:

```bash
node -v
npm -v
```

---

# 🚀 Step 5: Deploy Your Application

Clone repository:

```bash
git clone YOUR_REPOSITORY_URL
```

Install dependencies:

```bash
npm install
```

Run application:

```bash
npm start
```

For production deployment:

```bash
npm install -g pm2

pm2 start server.js
pm2 save
pm2 startup
```

---

# 🚀 Step 6: Create an AMI

1. Go to **EC2 → Instances**.
2. Select the configured instance.
3. Choose:

```text
Actions → Image and Templates → Create Image
```

4. Provide an image name.
5. Click **Create Image**.

The AMI stores:

* Operating System
* Installed Packages
* Application Code
* Server Configuration

---

# 🚀 Step 7: Launch Additional EC2 Instances

1. Open **EC2 → AMIs**.
2. Select the created AMI.
3. Click **Launch Instance from AMI**.
4. Select the same:

   * Key Pair
   * Security Group

Repeat to create multiple servers.

Example:

```text
Server-1
Server-2
Server-3
```

---

# 🚀 Step 8: Create a Target Group

1. Navigate to:

```text
EC2 → Target Groups
```

2. Click **Create Target Group**.

Configuration:

```text
Target Type : Instances
Protocol    : HTTP
Port        : 80
```

3. Register all EC2 instances.
4. Create the target group.

---

# 🚀 Step 9: Create an Application Load Balancer

1. Navigate to:

```text
EC2 → Load Balancers
```

2. Click **Create Load Balancer**.
3. Select:

```text
Application Load Balancer
```

4. Configure:

   * Name
   * Internet Facing
   * Availability Zones
   * Subnets
   * Security Group

5. Configure Listener:

```text
HTTP : 80
```

6. Attach the Target Group.
7. Click **Create Load Balancer**.

Wait until the status becomes **Active**.

---

# 🚀 Step 10: Verify Load Balancer

Open the DNS Name generated by AWS.

Example:

```text
my-load-balancer.ap-south-1.elb.amazonaws.com
```

Your application should now be accessible through the load balancer.

---

# 🚀 Step 11: Configure Route 53

## Create Hosted Zone

1. Open Route 53.
2. Navigate to **Hosted Zones**.
3. Click **Create Hosted Zone**.
4. Enter your domain name.

AWS generates Name Servers.

Update these Name Servers in your domain provider account.

Examples:

* Hostinger
* GoDaddy
* Namecheap

---

## Create Root Domain Record

Create a new record:

```text
Type  : A
Alias : Enabled
```

Select:

```text
Alias to Application Load Balancer
```

Choose:

* Region
* Load Balancer

Save the record.

---

## Create WWW Record

Create another record:

```text
Name  : www
Type  : CNAME
Value : yourdomain.com
```

---

# 🚀 Step 12: Request an SSL Certificate

1. Open AWS Certificate Manager (ACM).
2. Click **Request Certificate**.
3. Select:

```text
Request a Public Certificate
```

4. Add:

```text
yourdomain.com
www.yourdomain.com
```

5. Submit the request.

---

# 🚀 Step 13: Validate SSL Certificate

1. Open the certificate.
2. Click:

```text
Create Records in Route 53
```

3. AWS automatically creates validation records.

Wait until status becomes:

```text
Issued
```

---

# 🚀 Step 14: Enable HTTPS

1. Open the Load Balancer.
2. Go to **Listeners**.
3. Add:

```text
HTTPS : 443
```

4. Select:

   * Target Group
   * ACM Certificate

Save changes.

---

# 🚀 Step 15: Redirect HTTP to HTTPS

Edit the HTTP Listener Rule.

Configure:

```text
Action      : Redirect
Protocol    : HTTPS
Port        : 443
Status Code : 301
```

Save changes.

Now all traffic automatically redirects to HTTPS.

# 🧹 Cleanup Resources

To avoid AWS charges, delete resources in this order:

1. Load Balancer
2. Target Group
3. EC2 Instances
4. AMIs
5. Snapshots
6. Security Groups
7. Route 53 Records
8. Hosted Zone
